# 커밋 메시지 생성

스테이징된 변경사항을 분석하여 커밋 메시지 초안을 생성한다.
컨벤션은 `git-style` Rule을 따른다.

---

## 실행 절차

1. `git diff --staged`로 스테이징된 변경사항 확인
   - 스테이징된 파일이 없으면 커밋을 중단하고 아래 메시지 출력:
     ```
     ❌ 스테이징된 파일이 없습니다.
     커밋할 파일을 직접 git add 후 다시 실행해주세요.
     ```
2. 스테이징된 파일 목록을 출력하여 사용자에게 확인
3. 변경사항을 분석하여 적절한 type 선택
4. 변경 내용을 요약하여 제목 작성
5. 커밋 메시지 초안을 출력하고 사용자 확인을 받은 후에만 `git commit` 실행

> **IMPORTANT**: `git add`는 절대 실행하지 않는다. 사용자가 직접 스테이징한 파일만 커밋한다.
> **IMPORTANT**: 커밋 메시지에 `Co-Authored-By` 트레일러를 절대 추가하지 않는다.

---

## 출력 형식

```
<type>(변경된 부분 > 상세): 제목 (CC)
```

---

## 출력 예시

```
Design(Main > CongestionModal): Padding 값 조정 (CC)
```

---

## ✅ 검토 체크리스트

- [ ] type이 변경사항의 성격과 일치하는지 확인
- [ ] 제목이 변경 내용을 정확하게 요약하는지 확인
